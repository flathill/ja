説明

Proxmox VEのノード管理ツール(pvenode)では、ノード固有の設定やリソースを制御できます。

現在pvenodeでは、ノードの説明の設定、ノードのゲストに対するさまざまな一括操作の実行、ノードのタスク履歴の表示、pveproxy を介した API および Web GUI で使用されるノードの SSL 証明書の管理を行うことができます。

フォーマットオプション

出力形式は--output-formatパラメータを使用して指定できます。デフォルトのフォーマット・テキストは、ASCII-art を使用してテーブルの周りに美しいボーダーを描画します。さらに、いくつかの値を人間が読めるようなテキストに変換したりします：

  • Unix のエポックは ISO 8601 の日付文字列で表示されます。

  • 期間は週/日/時/分/秒のカウントで表示されます。

  • バイトサイズの値は、単位（B、KiB、MiB、GiB、TiB、PiB）を含みます。

  • 端数はパーセンテージで表示され、1.0は100%として表示されます。

また、--quietオプションを使えば、出力を完全に抑制することもできます。

--human-readable <boolean>(default = 1)

出力レンダリング関数を呼び出して、人間が読めるテキストを生成します。

--noborder <boolean>(デフォルト = 0)

ボーダーを描画しません（テキスト形式の場合）。

--noheader <boolean>(デフォルト = 0)

列ヘッダーを表示しません（テキスト形式の場合）。

--output-format <json | json-pretty | text | yaml>(default = text)

出力形式。

--quiet <ブール値

結果の印刷を抑制します。

例

外部から提供された証明書をインストールします。

pvenode cert set certificate.crt certificate.key -force

どちらのファイルもPEMエンコードされている必要があります。certificate.keyには秘密鍵が含まれ、certificate.crtには証明書チェーン全体が含まれます。

ACMEアカウントをセットアップし、ローカルノード用の証明書を注文します。
pvenode acme account register default mail@example.invalid pvenode config set --acme domains=example.invalid pvenode acme cert order systemctl restart pveproxy

ウェイクオンLAN

Wake-on-LAN (WoL)は、マジックパケットを送信することで、ネットワーク内でスリープしているコンピュータの電源を入れることができます。少なくとも1つのNICがこの機能をサポートし、コンピュータのファームウェア(BIOS/UEFI)設定でそれぞれのオプションを有効にする必要があります。オプション名はEnable Wake-on-LanからPower On By PCIE Device まで様々で、不明な場合はマザーボードのベンダーマニュアルを確認してください：

ethtool <interface> | grep Wake-on

pvenode は、WoL を介してクラスタのスリープメンバーを目覚めさせることができます：

pvenode wakeonlan <ノード

これは、wakeonlanプロパティから取得した<node>の MAC アドレスを含む WoL マジックパケットを UDP ポート 9 にブロードキャストします。ノード固有のwakeonlanプロパティは、次のコマンドで設定できます：

pvenode config set -wakeonlan XX:XX:XX:XX:XX

WoL パケットを送信するインターフェースはデフォルトルートから決定されます。これは、以下のコマンドでbind-interfaceを設定することで上書きできます：

pvenode config set -wakeonlan XX:XX:XX:XX:XX,bind-interface=<iface-name>.

WoLパケットを送信するときに使用されるブロードキャストアドレス（デフォルトは255.255.255.255）は、さらに以下のコマンドを使用して明示的にbroadcast-addressを設定することで変更できます：

pvenode config set -wakeonlan XX:XX:XX:XX:XX,broadcast-address=<broadcast-address>.

タスク履歴

バックアップジョブの失敗など、サーバーの問題をトラブルシューティングする場合、以前に実行したタスクのログがあると役立つことがよくあります。Proxmox VEでは、pvenode taskコマンドでノードのタスク履歴にアクセスできます。

listサブコマンドを使用すると、ノードの終了したタスクのフィルタリングされたリストを取得できます。例えば、エラーで終了したVM100に関連するタスクのリストを取得するには、次のようにコマンドを実行します：

pvenode task list --errors --vmid 100

タスクのログはUPIDを使って出力できます：

pvenode task log UPID:pve1:00010D94:001CA6EA:6124E1B9:vzdump:100:root@pam：

一括ゲスト電源管理

多数のVM/コンテナがある場合、ゲストの起動と停止は、pvenodeの startallサブコマンドとstopallサブコマンドで一括して実行できます。 デフォルトでは、pvenode startallはブート時に自動的に起動するように設定されたVM/コンテナのみを起動します（仮想マシンの自動起動とシャットダウン参照）。両コマンドには--vmsオプションもあり、停止/起動されるゲストを指定されたVMIDに制限します。

例えば、VM100、101、102を、オンブートが設定されているかどうかに関係なく起動するには、--vmsオプションを使用します：

pvenode startall --vms 100,101,102 --force

これらのゲスト（および実行中の他のゲスト）を停止するには、コマンドを使用します：

pvenode stopall
stopallコマンドはまずクリーンシャットダウンを試み、すべてのゲストが正常にシャットダウンされるか、オーバーライド可能なタイムアウト（デフォルトでは3分）が切れるまで待機します。これが発生し、force-stopパラメーターが明示的に0（false）に設定されないと、まだ実行中のすべての仮想ゲストがハード停止されます。

最初のゲストの起動遅延

VM/コンテナが起動の遅い外部リソース（NFSサーバなど）に依存している場合、Proxmox VEが起動してから、自動起動に設定されている最初のVM/コンテナが起動するまでにノードごとに遅延を設定することもできます（仮想マシンの自動起動と自動シャットダウンを参照）。

これを実現するには、次のように設定します(ここで10は秒単位の遅延を表します)：

pvenode config set --startall-onboot-delay 10

ゲストの一括移行

アップグレードの状況により、すべてのゲストを1つのノードから別のノードに移行する必要がある場合、pvenodeには一括移行のためのmigrateallサブコマンドも用意されています。デフォルトでは、このコマンドはシステム上のすべてのゲストをターゲットノードに移行します。しかし、ゲストのセットだけを移行するように設定することもできます。

たとえば、ローカルディスクのライブマイグレーションを有効にして、VM100、101、および102をノードpve2に移行するには、次のように実行します：

pvenode migrateall pve2 --vms 100,101,102 --with-local-disks。

バルーン用の RAM 使用量ターゲット

自動メモリ割り当ての目標パーセンテージのデフォルトは80%です。このターゲットは、ballooning-targetプロパティを設定することで、ノードごとにカスタマイズできます。たとえば、ホスト・メモリ使用率を90%に設定するには、次のようにします：

pvenode config set --ballooning-target 90
